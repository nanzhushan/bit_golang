// 无长度的通道用处，作用是step2要等step1执行完了才能执行，但是和开发工具顺序写代码不同的是这是多线程的。
// 无长度的通道是多线程的同步。
// 如果不用通道，就是但线程的同步
// 如果是定义有长度的通道，是多线程的异步
 
c = make(chan string)     // 定义无长度的通道
step1: 执行逻辑1  -> c
step2: <- c {
  执行逻辑2
}

make(chan string) // 没有定义长度的也就是无缓冲通道

goroutine1 order_id -> channel(100) // 可以多个协程往channel里写,这个是缓冲通道定死了长度

goroutine2  order_id <- channel （   // 可以多个协程从channel里取
  go func(order_id) {
    拿到order_id 逻辑入库    // 实际上是多协程在跑
   }
)

p1: 负责发送order_id // 如果没有往里面塞数据，P2就一直在等待
p2: 负责接受order_id // 每接受一个交给另外一个人去处理逻辑
